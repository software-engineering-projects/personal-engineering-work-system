# personal-engineering-work-system

📘 Personal Engineering Work System (PEWS) — Procedural Version

0. Core Rule

  - Work must be structured, scoped, and trackable.
  - No unrecorded or ad-hoc execution.

1. Task Intake (Capture Everything)

When anything appears (email, request, idea, issue):

  - Write it in your Task Inbox
  - Do NOT act immediately
  - Assign:
      - Type: Bug / Feature / Automation / Maintenance / Learning
      - Priority: P0 / P1 / P2 / P3

2. Task Clarification (Before Any Work)

For each selected task, define:

  - Goal: What needs to be achieved
  - Done Criteria: How you confirm completion
  - Inputs: What you need
  - Scope: What is NOT included

Note: If unclear → do NOT proceed.

3. Task Breakdown (Planning)

Convert task into steps:

1.  Step 1: Understand system/data
2.  Step 2: Identify logic/requirements
3.  Step 3: Implement solution
4.  Step 4: Test with real cases
5.  Step 5: Validate output
6.  Step 6: Deploy/finish

Note: If you cannot break it down → understanding is incomplete.

4. Execution (Focused Work Block)

Rules:

  - Work on ONE task only
  - No email/messages/side tasks
  - No multitasking
  - Use deep focus mode

Interruptions:

  - Write in Inbox
  - Return to current task only after block ends

5. Debugging Loop (When Issues Happen)

Follow strict cycle:

1.  Reproduce issue
2.  Inspect logs/output
3.  Isolate cause
4.  Form hypothesis
5.  Test fix
6.  Confirm resolution

Note: No guessing or random changes.

6. Validation (Definition of Done)

Before closing task:

  - Meets all done criteria
  - Handles expected edge cases
  - Output is verified
  - No unresolved errors

Note: If not → return to execution.

7. Task Closure

When complete:

  - Mark task as DONE
  - Record result briefly
  - Note any key learning or issue pattern

8. Interrupt Handling

When interrupted:

  - Do NOT switch immediately
  - Log into Inbox
  - Label urgency
  - Resume current task or stay idle until planned switch point

Rule:

  - Interruptions are recorded, not executed instantly

9. Daily Flow Structure

A. Intake Block

  - Review inbox
  - Prioritize tasks (P0–P3)
  - Select top task

B. Execution Block

  - Work on one task only
  - Deep focus mode

C. Maintenance Block

  - Emails, small fixes, admin work

10. Prioritization Rule

Always choose:

  - Highest impact + clearest definition first

Not:

  - Newest or most frequent request

Summary Mental Model

You operate as: Input → Clarify → Plan → Execute → Debug → Validate → Close

Not: React → Switch → Check → Repeat
